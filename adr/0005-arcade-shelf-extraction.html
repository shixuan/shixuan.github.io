ADR 0005 — Game Card 框架独立为 npm 包 arcade-shelf

Context

ADR 0004 里搭的 game-card 框架本来是博客内嵌的一坨东西：widgets/game-card.jsx + source/js/game-card.js + source/js/games/*.js + scripts/inject-css.js 自动扫描。能跑，但加完 minesweeper 之后有个不对劲的感觉 — 这套”自描述游戏 + SSR shell + 模态运行时”的结构跟博客本身没关系，任何装了 Hexo/Jekyll/11ty/纯 HTML 的地方都能用。继续塞在博客里相当于把一个能独立存在的工具和一个 content site 绑死。

所以抽出来，发成 npm 包 arcade-shelf — “a shelf for your mini games”。博客这边只保留壳和 init 代码，真正的 framework 从 unpkg CDN 拉。

Decision

arcade-shelf 用 Vite library mode 打包，同时产出 ESM (dist/index.js)、UMD (dist/index.umd.cjs)、rolled-up .d.ts、dist/style.css。公共 API：

核心改动是把原来的 window.GameRegistry queue-stub 替换成 per-shelf 的闭包 registry — ES module 天然解决了加载顺序问题（import 即同步可用），不再需要 global 占位。游戏契约 (Game 接口) 和 0004 基本一致：{ name, icon?, description?, order?, canvasSize?, footer?, init(canvas): GameHandle }，deltaTime 归一化的 Pong 和带难度的 Minesweeper 作为内置参考实现一起 ship。

博客这边的集成走 UMD + CDN 路线（Hexo 不是 bundler、也不想给博客多上一层构建）：

  1. widgets/game-card.jsx SSR 只产出 <div class="arcade-shelf-mount" data-title="..." data-whitelist="..." data-order="...">
  2. scripts/inject-css.js 从 unpkg 注入 arcade-shelf@<version>/dist/style.css 和 index.umd.cjs（带版本号 pin）
  3. source/js/arcade-shelf-init.js 扫所有 .arcade-shelf-mount，读 data-* 配置，createShelf().register(pong).register(minesweeper).mount()
  4. source/css/arcade-shelf-override.css 把 arcade-shelf 自己的 card 边框/背景置空，避免跟 Icarus .card.widget 产生双层卡片

升级 arcade-shelf：在 npm 上发新版，改 inject-css.js 里的 ARCADE_SHELF_VERSION 常量，重启 hexo server，完事。

Notes

为什么发包 vs git submodule vs 直接 copy

短期来看三种都能用，但发 npm 有两个长期好处：

  • 版本号就是文档 — arcade-shelf@0.1.1 这个字符串在 inject-css.js 里钉死，未来 npm registry 永远能 resolve 回当时那个 tarball。两年后回来调试，不用猜”当时用的是哪个 commit”。
  • unpkg 是免费的 CDN + 永久档案 — 只要 npm 不删包，https://unpkg.com/arcade-shelf@0.1.0/dist/... 永远可用。博客不用自己托管 asset，也不用担心 CDN 到期。

submodule 会让博客 repo 多一个 subtree 需要维护；直接 copy 没有版本化，upstream 改了不知道本地要不要同步。

SSR shell 为什么还留着

0004 的 SSR shell 做了两件事：(a) 渲染卡片 chrome（颜色、圆角、阴影），(b) 把 yaml 配置序列化成 data-*。

抽 arcade-shelf 之后 (a) 其实可以让 arcade-shelf 自己画，但博客这边的卡片 chrome 是 Icarus .card.widget 给的（要跟 profile card、subscribe card 视觉一致），没法让 arcade-shelf 替代。所以保留 SSR shell，让它只负责”产一个带 Icarus 外壳的挂载点”，arcade-shelf 在挂载点内部渲染 shelf 内容，外层用 override.css 把 arcade-shelf 的 card 自装饰擦掉。

换个没有主题的站点（纯 HTML、Jekyll 裸奔），可以完全跳过 SSR shell，直接写 <div id="games"></div> 让 arcade-shelf 自己画卡片。README 里有这种用法的示例。

defer 注入顺序

UMD 脚本必须比 init 脚本先执行，因为 init 脚本引用 window.ArcadeShelf。defer 保证了两件事：HTML 解析完才跑、按 DOM 出现顺序执行。所以 inject-css.js 里的注入顺序就是执行顺序：

文件删掉了啥

从博客 repo 删掉：

  • source/js/game-card.js — framework（现在在 arcade-shelf 里）
  • source/js/games/pong.js — 内置游戏（现在在 arcade-shelf 里）
  • source/js/games/minesweeper.js — 内置游戏
  • source/js/games/ — 整个目录
  • source/css/game-card.css — shelf + modal 样式（现在在 arcade-shelf dist/style.css）

新增：

  • source/js/arcade-shelf-init.js — 集成胶水
  • source/css/arcade-shelf-override.css — 卡中卡样式覆写

widgets/game-card.jsx 和 scripts/inject-css.js 保留但都精简了。

未来想加新游戏

两条路：

  • 发进 arcade-shelf — 写在 src/games/<name>.ts，PR 到 arcade-shelf repo，发新版，更新博客的 ARCADE_SHELF_VERSION。适合通用游戏。
  • 博客本地注册 — 在 source/js/arcade-shelf-init.js 里定义 const myGame = { name, icon, init, ... }，然后 shelf.register(myGame)。完全不用动 arcade-shelf。适合跟博客内容耦合的游戏（比如”扫雷生成的雷区对应当天的博文数”这种奇葩玩法）。

preview_count + More overlay — sidebar 空间管控

arcade-shelf 0.2 开始有三个内置游戏（pong / minesweeper / snake），以后大概率还会更多。但 Icarus 左栏就那么点宽，把所有游戏塞进 sidebar 会把 subscribe / toc 等 widget 挤下去。所以引入”紧凑模式”：sidebar 只显示前 N 个，底下一个 “more…” 按钮，点开浮层列出全部。

核心约束是 arcade-shelf 不动。这个需求本质是”侧栏宽度受限”，是博客这一端的布局关切 —— ADR 0005 的前提就是 arcade-shelf 不该知道 Hexo / Icarus 存在。所以全部逻辑做在 init.js + override.css 里。

.filter(Boolean) 是为了把版本 gate 做掉：博客 pin 在 0.1.x 时 AS.snake 是 undefined，自动被过滤；bump 到 0.2 之后它自己出现，init.js 完全不用改。

Single-modal UX：点 more → 浮层列全部 → 点游戏 → 先拆掉浮层，再开游戏 modal。不 stack。关键是用 arcade-shelf 的公共导出 openGameModal(game, triggerBtn)：

这条路径不碰 arcade-shelf 内部的 list click dispatcher，走的是 library 明确 export 的入口 —— 等于把博客当成一个”composing consumer”，跟 React 用户写 openGameModal 打开游戏没任何区别。如果做两个 arcade-shelf 实例叠起来然后 intercept click，就会陷入”蜡烛两头烧”：既要懂 arcade-shelf 内部 event 路径，又要保证未来版本升级时它还能工作。

more 按钮的 SSR + 渐进增强：

hidden 属性是关键。按钮在 HTML 里就有，但 JS 挂载成功之后才 unhide。两个好处：

  • JS 失败 / arcade-shelf UMD 没加载，按钮不会挂尸在页面上让用户点了没反应。
  • 有剩余游戏（sorted.length > previewCount）时才 unhide，没有时保持隐藏 —— 博客 pin 在 0.1.1 时 snake 没出现，2 > 2 false，more 按钮不显示；bump 到 0.2.0 后 3 > 2 true，自动出现。这也是个优雅降级点。

Overlay 自己写，不复用 arcade-shelf 的 modal：因为 arcade-shelf 的 modal API (openGameModal) 是”给你一个游戏、我帮你开一个 canvas”，不是”列一堆游戏让用户选”。语义不对。所以 overlay 是博客手写的 ~100 行：<ul> + <li> + 背景、Esc、backdrop click、close 按钮、document.body.style.overflow lock 这些常规模态窗逻辑。深色模式走 @media (prefers-color-scheme: dark)，跟 arcade-shelf 自身的配色 philosophy 对齐。

Implementation Sketch

(as of 2026-04-21，会随代码漂移 — 文件本身是 canonical)

博客里跟 arcade-shelf 相关的全部文件：

  • widgets/game-card.jsx — Inferno SSR，吐 <div class="arcade-shelf-mount" data-*> + 条件 <button class="game-card-more-btn" hidden>
  • scripts/inject-css.js — ARCADE_SHELF_VERSION 常量 + unpkg 链接 + init script 注入
  • source/js/arcade-shelf-init.js — 扫 mount 元素、按 order 取 top N、挂 shelf、绑 more 按钮、render overlay、单 modal 切换逻辑
  • source/css/arcade-shelf-override.css — shelf 视觉融合覆写 + more 按钮 + overlay 全套样式（含 dark mode）
  • _data/shared-widgets.yml — 配置入口（title/whitelist/order/preview_count）

arcade-shelf 包本身的代码在 github.com/shixuan/arcade-shelf，README 里有公共 API、游戏契约、集成 recipes（React/Hexo/Jekyll/纯 HTML）。
